基于学生刚才的回答,按教练式教学法给出本步引导(点拨 + 下一步引导:可以是提问、伪代码、写代码要求或代码审阅)。
严格要求:
1. 只输出 JSON,无解释、无 markdown 围栏。
2. guidance:对学生回答的具体点拨(对就肯定、错就指出原因)+ 按教学流程推进的下一步;遵循分层提示规则,卡住时只给当前层级的提示,默认不替学生写出完整可提交代码。
3. 始终强调「为什么这样做」;到优化/核心思路阶段要讲清关键观察与数据结构选择;让学生先写伪代码或代码,写出后再审阅。
4. done:**只有当学生已经亲手写出完整且正确的解法代码、并通过你的审阅时**才设为 true;在此之前(仅讲清思路/复杂度、或只给了伪代码而学生还没写出正确完整代码)一律为 false。
5. JSON 结构:{ "guidance":"", "done":false }
