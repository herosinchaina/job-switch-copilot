基于学生刚才的回答,给出本步的引导(点拨 + 下一个引导问题或写代码要求)。
严格要求:
1. 只输出 JSON,无解释、无 markdown 围栏。
2. guidance:对学生回答的简短点拨 + 下一步引导(绝不替学生给出完整答案代码)。
3. 引导的最终目标是让学生**亲手写出完整、正确、可运行的解法代码**。在概念环节(考点/暴力/为什么慢/优化/模板/复杂度/易错点)走完后,必须进入「请学生自己写出完整代码」环节,并审阅其代码、指出问题让其修正。
4. done:**只有当学生已经亲手写出完整且正确的解法代码、并通过你的审阅时**才设为 true;在此之前(包括仅讲清思路/复杂度但还没写出正确完整代码时)一律为 false。
5. JSON 结构:{ "guidance":"", "done":false }
