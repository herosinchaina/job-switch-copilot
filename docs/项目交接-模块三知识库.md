## 一、项目定位与架构(已定型,勿改)

**产品**:AI 求职操作系统(个人自用真实产品,方案 A)。需求文档:`data/AI_Interview_OS_Detailed_Product_Prompt.docx`(已 gitignore,本地在 `data/`)。

**架构**:
```
React 前端 (Vite+TS+Tailwind)  ←HTTP+SSE→  本地 Node 后端 (Express)  ←→  claude CLI 子进程
                                              SQLite (node:sqlite, --experimental-sqlite)
```
- **monorepo**:`apps/web` + `apps/server` + `packages/shared`(共享 TS 类型/zod 模型)
- **AI 引擎**:全部走 `AiProvider` 适配层 → `ClaudeCliProvider`(`spawn('claude', [...])`,`shell:false`)。复用本机 Claude CLI,不填 API Key。留了口子将来加 `ApiKeyProvider`。
- **多轮会话**:`ClaudeCliProvider` 支持 `startSession/continueSession`(用 `--session-id`/`--resume`),DB 存权威 Q&A;`--resume` 失败自动降级到"拼历史"无状态模式。
- **AI 老师(仅 LeetCode 引导)固定 `--model sonnet`**,其余用 CLI 默认模型。`getGuideAi()` 单例注入 `leetcodeRouter`。

**三大硬约束(用户反复强调)**:流畅性、稳定性、安全性优先。
- 安全:`shell:false` 数组传参、仅绑 `127.0.0.1`、上传限制、AI 输出 zod 校验、SQL 参数化、限并发队列(≤2)、CLI 调用超时 kill、CLI 启动自检横幅。
- 稳定:AI 非法 JSON → 重试 → 降级报错,不静默崩;SQLite 写用事务。

**关键 DB schema**:`apps/server/src/db/connection.ts`;repo 在 `apps/server/src/db/repo.ts`。差题打标字段 `is_weak`(interview_turns 和 project_deepdive_turns 都有)。

## 四、下一步:模块三 知识库(设计已敲定,待写 spec)

**定位**:长期知识积累中心。消费模块四/五沉淀的 `is_weak` 差题 + 手动新增 + 艾宾浩斯复习。冷启动已解决(模块四/五在打标差题)。

**已确认的设计决策(用户已同意方案 A + 统一表 + 自评复习)**:

- **范围 = A**:消费差题 + 手动新增 + 复习。**不做**独立主动出题引擎(与模块四/五重叠)。重心是**沉淀 + 复习**。
- **数据模型 = 统一 `knowledge_items` 表**(不是只读视图,因为艾宾浩斯复习状态必须可持久化):
```
knowledge_items
  id, question, answer(用户原答,可空), reference(标准/更优答法),
  tags(知识点标签,JSON 或逗号分隔), source('interview'/'deepdive'/'manual'),
  source_ref(来源 turn id,可空), note(手动补充,Markdown),
  mastery(0未掌握~5已掌握),
  review_due(下次复习时间), review_interval(间隔天数), review_count(复习次数),
  created_at, updated_at
```
- **最小闭环范围**:
  1. **导入差题**:模块四/五结束后把 `is_weak` 问题存入知识库(手动按钮或 API);读 `interview_turns`/`project_deepdive_turns` 的 is_weak 记录。
  2. **手动新增**:问题+答案+标签+Markdown 笔记。
  3. **列表+搜索**:按标签/来源/掌握度筛,关键词搜(SQLite LIKE 起步)。
  4. **艾宾浩斯复习**:每项有 `review_due`;"今日复习"列到期项;复习流程 = 展示问题→回忆→看答案→**自评"记住了/没记住"**→答对拉长间隔(1→2→4→7→15天…)、答错重置。**基础间隔算法,不做 AI 评分复习**(轻量)。
  5. **标签归类**:手动标签先行,**AI 自动聚合到知识点树后置**。
- **已确认**:复习用**自评驱动间隔,不做 AI 评分**(用户在原对话回了"同意你的方案"确认此点)。

**下一步动作**:直接进 writing-plans 把上述设计写成 spec(`docs/superpowers/specs/2026-07-0X-knowledge-base-design.md`)→ plan → subagent 执行。设计已完整,无需再问用户,可直接写 spec 给用户过目。
