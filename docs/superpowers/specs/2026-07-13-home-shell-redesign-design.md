# 首页与壳层重设计:暗色作战台 · 模块大厅

- 日期:2026-07-13
- 状态:待评审
- 范围:前端壳层(侧栏导航)、设计系统令牌、首页「模块大厅」、以及后续子页统一骨架与分批 IA 重做。本规格批准后先落地 Phase 0;子页深度重排按 Phase 1/2 另开实现计划。

## 1. 定位

「AI 求职操作系统」首页的首要工作是**功能大厅**:用户打开即看到全部模块入口,美观、可点、有科技感。准备度等数据只做轻量点缀,不抢主叙事。

当前痛点:全部功能挤在顶栏,首页(Dashboard)以大号准备度环为主,模块感弱、视觉普通。

## 2. 已锁定决策

| 项 | 决定 |
|---|---|
| 首页职责 | 模块大厅(统计次要) |
| 导航 | 左侧固定侧栏 |
| 视觉 | 暗色作战台(深空 + 香槟金/青冷点缀) |
| 首页布局 | B 修订:默认均等宫格;悬停哪个放大哪个(非固定主推块) |
| 改动深度 | 全站视觉 + 子页信息架构重做 |
| 推进 | 分阶段:先系统+壳+大厅,再分批子页 |

## 3. Phase 0 范围(本期实现)

**做:**

1. 设计系统 CSS 变量与字体(替换 Inter 默认栈)
2. `App` 壳层:顶栏导航 → 左侧侧栏 + 细顶栏
3. 新首页「模块大厅」(默认落地页,不再默认进简历)
4. 准备度并入大厅顶栏轻量显示;原独立 Dashboard 大 Hero 降级/并入,不再作为默认首页主视觉
5. 主题切换、导出数据迁到侧栏底部(或细顶栏次要区)

**不做(Phase 0):**

- 各子页 IA 深度重排(仅套上新壳与令牌,布局大体保留)
- 新后端 API
- 营销向落地页 / 未登录态

## 4. 信息架构 · 壳

```
┌──────────────┬─────────────────────────────────────┐
│ Brand        │  细顶栏: 当前模块名 / 面包屑         │
│ 模块大厅     │─────────────────────────────────────│
│ 简历大师     │                                     │
│ 模拟面试     │           主内容区                  │
│ 项目深挖     │                                     │
│ 知识库       │                                     │
│ 错题本       │                                     │
│ 算法学习     │                                     │
│ ─────────    │                                     │
│ 主题 / 导出  │                                     │
└──────────────┴─────────────────────────────────────┘
```

- 侧栏宽度约 240–248px;移动端可折叠为抽屉(Phase 0 需可用的最小响应式)。
- `view` 状态机保持现有联合类型;新增或将 `dashboard` 语义改为 `home`(模块大厅)。推荐:`home` 为默认;`dashboard` 路由别名指向 `home` 或删除独立 Dashboard 页。
- 侧栏当前项高亮用香槟金浅渐变 + 细边框(对齐现有 prototype)。

## 5. 首页 · 模块大厅

### 5.1 结构

- 顶行:标题「模块大厅」+ 右侧 `准备度 NN%`(等宽数字,非大环 Hero)
- 主体:6 模块均等宫格(3×2 桌面;窄屏 2 列)
  - 简历大师 / 模拟面试 / 项目深挖 / 知识库 / 错题本 / 算法学习
- 每格:模块名、底部细进度条(有数据则填;无则淡空)、悬停时展开一行状态文案

### 5.2 签名交互:悬停放大

- **默认**:各 tile 同等尺寸,无「固定主推」大块
- **悬停 / 键盘焦点**:当前 tile `scale(~1.08)`,边框与背景吃模块色调;一行 hint 展开;其余 tile 略降透明度
- **不**用固定「今日推荐」占一大格来制造层级
- `prefers-reduced-motion: reduce` 时取消 scale,仅改边框/背景
- 触控:无 hover 时依赖 `:active` / 焦点态等价反馈;点击仍直接进入模块

### 5.3 数据

复用现有 Dashboard stats API / `computeReadiness` 逻辑;大厅只消费摘要字段做进度条与顶栏百分比。加载中用骨架;失败时顶栏显示「准备度 —」,宫格仍可导航。

## 6. 视觉令牌

### 6.1 Color

| 名 | Hex | 用途 |
|---|---|---|
| Void | `#0A0E1A` | 页面底 |
| Panel | `#121826` / `rgba(255,255,255,.03–.05)` | 侧栏与卡片 |
| Signal | `#EEF1F7` | 主文字 |
| Muted | `#98A0B3` | 次文字 |
| Faint | `#5F6880` | 标签/弱信息 |
| Champagne | `#D9B877` | 主强调 / 激活 |
| Accent-2 | `#E8CF9A` | 强调高光 |
| Cyan | `#7FD4D0` | 模块点缀(面试等) |
| Violet | `#A99BE0` | 模块点缀(深挖等) |
| Emerald | `#7ECFA0` | 正向/知识 |
| Rose | `#E29AA4` | 警示/错题 |

背景氛围:径向深蓝光晕 + 香槟金弱光 + 细点阵;避免紫霓虹描边与多层 glow 堆砌。

浅色模式:Phase 0 保留切换能力,但作战台以暗色为设计源;浅色为降饱和派生,不单独再做一套构图。

### 6.2 Type

- Display: **Space Grotesk**(模块名、大数字);中文回落 PingFang SC / 系统黑体
- Body: 系统 UI + PingFang SC
- Mono: **IBM Plex Mono**(准备度、进度、标签码)
- **弃用**全局 Inter 作为品牌字体

### 6.3 Motion / Signature

1. Hover Lift(首页签名)
2. Atmosphere(深空光晕 + 点阵)
3. Soft Scan(进度条偶发轻扫描高光,表示在线)

子页卡片沿用更克制的 hover lift(约 1.02–1.03),不做首页级放大。

## 7. 子页统一骨架(Phase 1+ 强制)

所有功能子页共用:

1. 面包屑:`模块 / 当前页`
2. 页头:标题 + 一句说明 + **右上唯一主 CTA**
3. 主体默认 **主工作区(左) + 上下文(右:状态/建议/历史)**;窄屏改为上下堆叠
4. 一页一事:页头 CTA 与主区主任务一致

## 8. 分批计划

| 阶段 | 内容 | 验收 |
|---|---|---|
| Phase 0 | 令牌 + 侧栏壳 + 模块大厅 | 默认进大厅;悬停放大;全模块可导航;主题/导出可用 |
| Phase 1 | 简历大师全链路 + 模拟面试 IA 重排 | 符合 §7 骨架;主 CTA 清晰 |
| Phase 2 | 项目深挖 / 知识库 / 错题本 / 算法学习 | 同上 |

本文件是 Phase 0 的实现依据;Phase 1/2 在开工前可各补一页短 spec 或直接写入对应 implementation plan。

## 9. 前端落点(实现时)

主要触及(预期):

- `apps/web/src/App.tsx` — 壳与导航
- `apps/web/src/index.css` — 令牌、字体、氛围、motion
- `apps/web/src/pages/Dashboard.tsx` 或新建 `Home.tsx` — 模块大厅
- `apps/web/src/components/ui.tsx` — Button/Card 对齐新令牌
- 可选:`apps/web/prototype/home-redesign.html` 仅作参考,不作为运行时依赖

保持现有 view 状态与各 page 组件 props 契约,避免 Phase 0 改业务逻辑。

## 10. 非目标与约束

- 不引入 React Router(除非后续单独立项)
- 不在 Phase 0 重写子页业务流
- 不新增后端字段;准备度算法沿用现有
- 可访问:侧栏与 tile 支持键盘焦点;尊重 reduced motion
- 测试:更新/补充壳与大厅相关组件测试;现有 page 测试在 Phase 0 应仍通过

## 11. 成功标准

1. 打开应用直接进入模块大厅,不再被顶栏七个按钮淹没
2. 视觉一眼可辨「暗色作战台」,且与现有 HTML prototype 气质一致但不照搬其固定大 Hero
3. 悬停放大是可感知的签名交互,而非某个模块永久变大
4. Phase 0 合并后,现有子页功能不回归
